# VPC Network Access Analysis using AWS

## Overview

Analyzed and validated VPC network configurations using AWS Network Access Analyzer to detect unintended access paths and strengthen cloud security posture.

## Objective

* Analyze and validate VPC network access paths
* Detect unintended or insecure connectivity
* Improve network security governance
* Ensure adherence to security best practices

## Problem Statement

* Misconfigured routing or security rules can expose sensitive resources
* Manual inspection of network paths is time-consuming and error-prone
* Lack of visibility into resource reachability increases security risk

## Architecture

* AWS Network Access Analyzer evaluates VPC configurations and policies
* Analyzes reachability between resources such as EC2 instances and subnets
* Identifies allowed and unintended access paths
* Helps enforce secure network design using Security Groups and NACLs

## AWS Services Used

* Amazon VPC
* AWS Network Access Analyzer
* Amazon EC2
* Security Groups
* Network ACLs
* Route Tables

## Outcome / Business Impact

* Prevented accidental exposure of resources
* Improved security compliance and governance
* Enabled faster identification of misconfigurations
* Enhanced visibility into network access paths
